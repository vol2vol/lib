# Online Library
Назначение программы – автоматизация процессов хранения, поиска, обработки и предоставления пользователями электронных информационных ресурсов (книг, учебных пособий, научных 
статей и других).

Область применения системы – ФГБОУ ВО “ИрНИТУ”. Система может использоваться студентами, преподавателями, научными сотрудниками и административным персоналом для оперативного доступа к информационным ресурсам, а также для управления библиотечным фондом в электронном виде.
## Функции
1. Формирование и ведение базы данных электронных ресурсов.
2. Обеспечение поиска ресурсов по заданным пользователем критериям.
3. Организация разграниченного доступа пользователей к ресурсам системы.
4. Обеспечение возможности добавления, редактирования, удаления и просмотра ресурсов.
5. Регистрация и учет действий пользователей.

## Инструкция по работе с проектом
Данный раздел содержит различные пояснения по развертыванию проекта и работе с ним
### Развертывание
1. Когда работаем в ветке **frontend**:
   - клонируем репозиторий, переходим в ветку frontend
   - запускаем Docker Desktop
   - в терминале в корне репозитория пишем команды:

     wsl

     docker-compose up -d node
     
   - frontend доступен по адресу: http://localhost:5173
2. Когда работаем в ветке **backend**:
   - клонируем репозиторий, переходим в ветку backend
   - запускаем Docker Desktop
   - в терминале в корне репозитория пишем команды:

     wsl

     docker-compose up -d laravel nginx
     
   - backend доступен по адресу: http://localhost:8000

3. Когда работаем в ветке **dev** (main):
   - клонируем репозиторий, переходим в ветку dev
   - запускаем Docker Desktop
   - в терминале в корне репозитория пишем команды:

     wsl

     docker-compose up -d
     
   - backend доступен по адресу: http://localhost:8000
     
     frontend доступен по адресу: http://localhost:5173

4. Развертывание может занять некоторое время. Обычно это происходит быстро, но может затянуться до минут 5.

### Работа в ветках
При любом изменении в ветке backend автоматически запускаются тесты, которые могут не отработать. Поэтому рекомендуется делать ответвление от ветки backend для внесения любых изменений, далее делать pull request в ветку backend. В случае отработки всех тестов - ветки можно сливать.
